  <name>Task 1: locations.js — Pointer Events drag engine IIFE</name>
  <files>tap-to-vocab/assets/js/locations.js</files>
  <action>
Create `tap-to-vocab/assets/js/locations.js` as an IIFE that exports `window.LocationsGame` with two public methods: `init(draggableEl, onDrop)` and `resetDraggable(el)`.

Implement the full drag loop using the exact pattern from 08-RESEARCH.md "Complete Minimal Drag Engine" section. Key implementation requirements:

**Module-level state variables:**
- `shiftX`, `shiftY` — grab offset (pointer position minus element corner on pointerdown)
- `originLeft`, `originTop` — element viewport position recorded on pointerdown for snap-back
- `snapTimer` — reference to the snap-back setTimeout so it can be cancelled
- `onDropCallback` — function registered via init()

**init(draggableEl, onDrop):**
- Store onDrop as onDropCallback
- Attach `pointerdown`, `pointermove`, `pointerup`, `pointercancel` to draggableEl
- Set `draggableEl.ondragstart = () => false` (disables native image drag on desktop Chrome)

**onPointerDown(e):**
- Guard: `if (!e.isPrimary) return` — ignore secondary touches (multi-touch)
- Cancel any pending snapTimer with clearTimeout
- Record grab offset: `shiftX = e.clientX - rect.left`, `shiftY = e.clientY - rect.top`
- Record origin: `originLeft = rect.left`, `originTop = rect.top`
- Set `el.style.position = 'fixed'`, `zIndex = '1000'`, `transition = 'none'`
- Set `left = (e.clientX - shiftX) + 'px'`, `top = (e.clientY - shiftY) + 'px'`
- Call `el.setPointerCapture(e.pointerId)` — routes all future pointer events here

**onPointerMove(e):**
- Guard: `if (!e.isPrimary) return`
- Guard: `if (!el.hasPointerCapture(e.pointerId)) return`
- Update `left` and `top` with current pointer minus shift
- Zone highlight: set `el.hidden = true`, call `document.elementFromPoint(e.clientX, e.clientY)`, set `el.hidden = false` — this is REQUIRED because the draggable is on top and would otherwise shadow the zone
- Use `.closest('[data-zone]')` on the result to find the zone element
- Use `classList.toggle('zone-hover', z === zone)` on all `[data-zone]` elements

**onPointerUp(e) — also used for pointercancel:**
- Guard: `if (!e.isPrimary) return`
- Guard: `if (!el.hasPointerCapture(e.pointerId)) return`
- Call `el.releasePointerCapture(e.pointerId)`
- Clear all zone-hover classes
- Run hide/elementFromPoint/unhide to detect final zone
- If zone found: call `onDropCallback(zone.dataset.zone, el)`
- If no zone: call `snapBack(el)`

**snapBack(el):**
- Set `el.style.transition = 'left 0.25s ease, top 0.25s ease'`
- Set `left = originLeft + 'px'`, `top = originTop + 'px'`
- Set `snapTimer = setTimeout(260ms, () => clear position/transition/zIndex/left/top styles)`

**resetDraggable(el):**
- clearTimeout(snapTimer), set snapTimer = null
- Clear all inline position styles: position, transition, zIndex, left, top

Do NOT add any game logic, exercise constants, or SharedUtils/CoinTracker calls — those are Phase 10 concerns.

  document.getElementById('draggable'),
