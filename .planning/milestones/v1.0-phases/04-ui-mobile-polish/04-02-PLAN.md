<task type="auto">
  <name>Task 3: Fix fill-blank.html h1 margin and add narrow-grid media query</name>
  <files>tap-to-vocab/fill-blank.html, tap-to-vocab/assets/css/styles.css</files>
  <action>
**Fix 1 — fill-blank.html h1 margin:**

In fill-blank.html, find the header flexbox row (line ~18):
```html
&lt;div style="display:flex; align-items:center; justify-content:space-between; gap:10px; flex-wrap:wrap;"&gt;
  &lt;h1&gt;✏️ Fill in the Blank&lt;/h1&gt;
```

Add `style="margin:0"` to the h1:
```html
  &lt;h1 style="margin:0"&gt;✏️ Fill in the Blank&lt;/h1&gt;
```

The h1 default in styles.css has `margin: 0 0 12px`. Inside a flexbox header row, this creates a 12px gap at the bottom of the h1 that pushes the layout off. The fix matches the pattern used by other pages where h1 is inside a flex row.

**Fix 2 — narrow-grid media query in styles.css:**

At the end of the `@media (max-width: 600px)` block (or as a separate new block after it), add:
```css
@media (max-width: 420px) {
  .grid-two-col .btn {
    font-size: 0.85rem;
    padding: 10px 8px;
  }
}
```

This reduces font size and horizontal padding for category buttons in the 2-column home grid at narrow viewports (375px). The label text (e.g. "🏠 Casa y Familia") will wrap less aggressively at ~146px column width. Min-height is NOT added here — the .btn class has padding:10px, giving ~42px which is acceptable for these nav buttons (not learning interactions).
  </action>

  </verify>
