<objective>
Fix the browse mode button layout regression on topic.html introduced in v1.1.

Purpose: v1.1 applied `flex-wrap: nowrap` globally to `.controls` inside `@media (max-width: 600px)` to prevent sentences/conjugation nav rows from wrapping. This unintentionally prevents the 6 browse mode buttons from wrapping to 2 rows on 375px viewports, causing overflow/clipping.

Output: Two-file change — HTML reorders buttons and adds a `browse-controls` class; CSS overrides `flex-wrap` back to `wrap` for that class.
</objective>

Replace this single rule with two rules:

```css

  flex-wrap: nowrap;

.browse-controls {
  flex-wrap: wrap;
}
```
