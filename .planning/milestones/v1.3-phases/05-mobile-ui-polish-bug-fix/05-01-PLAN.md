    Change the Home button text from `🏠 Home` to `🏠` (icon only):
    ```html

    <!-- After -->
    ```

    Add `title="Home"` so screen readers and pointer users know the purpose.

    **In styles.css:**
    The `.controls` class already has `flex-wrap: wrap`. On 375px with 4 buttons, they must fit on one row. The icon-only Home button is narrower which helps, but also add a scoped CSS rule that prevents wrapping in the controls row when it is inside a sentences/conjugation page context. Add this at the end of the existing `@media (max-width: 600px)` block:

    ```css
    /* Nav row: prevent wrap on narrow mobile — icon-only Home ensures fit */
    .controls {
      flex-wrap: nowrap;
    }
    ```

    This is safe because each button already has `padding: 10px 12px` at mobile size (from the existing `.btn` mobile override). Four buttons at ~80-90px each = ~320-360px which fits within 375px card content width (375 - 2*16px container padding - 2*20px card padding = 303px usable — actually the card has `padding: 20px` and `.container` has `padding: 16px`, so usable ≈ 375 - 32 - 40 = 303px).

    Re-check the math: at `@media (max-width: 600px)`, `.btn` has `padding: 10px 12px` and `font-size: 0.95rem`. Each button text:
    - "← Prev" ≈ 66px
    - "🔄 Reset" ≈ 72px
    - "Next →" ≈ 66px
    - "🏠" (icon only) ≈ 44px (min-height applies, width will be ~44px)

    Total ≈ 66+72+66+44 = 248px + 3 gaps×6px = 18px = 266px — fits in 303px. Safe.

    If buttons still overflow due to font rendering variance, add `font-size: 0.85rem` specifically to `.controls .btn` inside the mobile media query — but only do this if the icon-only change alone is insufficient. Prefer the minimal change first (icon-only + flex-nowrap).
