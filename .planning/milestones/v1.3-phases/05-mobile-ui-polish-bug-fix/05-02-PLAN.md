From tap-to-vocab/assets/js/conjugation.js (switchMode function, lines 356-377):
```javascript
function switchMode(mode) {
  if (mode === currentMode) return;
  currentMode = mode;
  document.querySelectorAll(".mode-tab").forEach(t => t.classList.remove("active"));
  if (mode === "practice") {
    tabPractice.classList.add("active");
    practiceControls.style.display = "";
    showControls.style.display = "none";
    if (!practiceInitialized) {
      initConjugationGame(getActiveVerbs());
      practiceInitialized = true;
    }
  } else {
    tabShow.classList.add("active");
    practiceControls.style.display = "none";
    showControls.style.display = "";
    if (!showInitialized) {
      initShowMode(getActiveVerbsOrdered());
      showInitialized = true;
    }
  }
}
```

Bug: `showInitialized` is never reset to false when switching away from Show. So on the second Show visit, `initShowMode` is not called, leaving the DOM in whatever state `initConjugationGame` left it (blank slots, used word bank).

Fix: Always call `initShowMode` when switching to show (remove the `showInitialized` guard entirely for show-mode switching), OR reset `showInitialized = false` when switching to practice mode.

  <name>Task 2: Fix Show mode blank screen bug in conjugation.js (BUG-01)</name>
  <files>tap-to-vocab/assets/js/conjugation.js</files>
  <action>
    **Root cause:** In `switchMode()`, `showInitialized` is only set to `true` and never reset to `false`. When the user goes Practice→Show→Practice→Show, on the second Show visit `showInitialized` is still `true` so `initShowMode` is never called. However, `initConjugationGame` was called when returning to Practice, which rebuilt the table DOM with empty slots and cleared the word bank. Show mode then shows that stale DOM.

    **Fix:** Remove the `showInitialized` guard entirely. Always call `initShowMode` when switching to Show mode. Show mode is read-only and lightweight (no timers, no complex state), so re-initializing it every time is safe and correct.

    In `conjugation.js`, in the `switchMode` function, change the `else` branch from:
    ```javascript
    } else {
      tabShow.classList.add("active");
      practiceControls.style.display = "none";
      showControls.style.display = "";
      if (!showInitialized) {
        initShowMode(getActiveVerbsOrdered());
        showInitialized = true;
      }
    }
    ```
    to:
    ```javascript
    } else {
      tabShow.classList.add("active");
      practiceControls.style.display = "none";
      showControls.style.display = "";
      initShowMode(getActiveVerbsOrdered());
      showInitialized = true;
    }
    ```

    Also remove the `showInitialized` variable declaration (`let showInitialized = false;`) since it is now only ever set but never read. Keeping it is harmless but removing it is cleaner. If removal causes any lint concern, leave it — correctness is the priority.

    Do NOT change `practiceInitialized` logic — Practice mode has stateful timers and history, so lazy-init there is intentional. Only Show mode gets the always-reinit treatment.
